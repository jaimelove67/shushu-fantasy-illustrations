# 鼠鼠幻想插图

## 核心定位

为中文社交平台内容生成“鼠鼠的幻想”风格插图：真实照片或白底画面上，贴一个低清、荒诞、土味可爱的鼠鼠角色，角色脑袋里冒出手绘幻想云，云里放它想象的对象、身份、奖励、恋爱、毕业、工作或发财场景。

目标是表情包式幽默，不是商业插画、精致 IP、PPT 信息图或完整漫画。画面要像手机里随手 P 出来的二创梗图：粗糙但明确，低成本但有记忆点。

## 先读这些参考

按任务需要读取，不要一次加载全部：

- `references/style-dna.md`：整体视觉 DNA、画面质感、颜色和禁忌。
- `references/reference-lock.md`：最关键的参考图硬约束。生成不贴近参考图时必须先读它。
- `references/shushu-ip.md`：鼠鼠角色设定、表情、身体、道具和动作。
- `references/fantasy-bubbles.md`：幻想泡泡类型、构图模式和云泡泡规则。
- `references/prompt-template.md`：单张生图和改图提示词模板。
- `references/qa-checklist.md`：生成后检查和重生成规则。
- `assets/examples/`：用户提供的三张参考图。生成时若图像工具支持参考图输入，必须附带至少 `02-coin-fantasy-template.png` 作为角色/泡泡风格参考；真实场景叠加时再附带 `01` 或 `03` 作为照片拼贴参考。不要逐像素复刻，除非用户明确要求“照这张做同款”。

## 工作流

### 1. 判断任务类型

先判断用户要哪一种：

- **真实照片叠加**：用户给了校园、办公室、实验室、公园、旅行照等背景，需要把鼠鼠贴进去并加幻想泡泡。
- **白底模板**：像表情包底图，白背景，鼠鼠在左下或下方，右上/上方一个幻想云。
- **配图策略**：用户给一个主题、文案或账号内容，要先规划几张鼠鼠幻想图。
- **提示词输出**：用户只要 Midjourney / DALL-E / image_generation 提示词。
- **改图**：用户要换泡泡内容、加道具、改字、把某个幻想换成另一个。

### 2. 提炼幻想点

每张图只抓一个幻想点：

- 想要的对象：恋爱、合照、被喜欢、收到花。
- 想要的身份：毕业生、工程师、研究员、摄影师、富豪、老板、上岸人。
- 想要的奖励：金币、证书、offer、奖杯、token、工资、假期。
- 想要的场景：汇报成功、实验室发光、旅行出片、课堂逆袭、项目过审。
- 想要的状态：被夸、被抱、摆烂成功、偷偷发财、被全场认可。

幻想要浅显，1 秒能懂。不要把笑点解释成流程图。

### 3. 设计画面

默认结构：

- 鼠鼠放在下方或画面近景，像被贴上去的贴纸。
- 幻想云从鼠鼠头顶冒出，主云占画面约 25%-45%。
- 云里只放 1 个主幻想，最多 2-3 个辅助元素。
- 中文文字默认可以没有；需要时只用“嘻嘻”“嘿嘿”“OAO”“好耶”等情绪词。不要把核心幻想写成解释性文字，例如“我是汉密尔顿”“我发财了”“我是博士”。
- 真实背景要保留环境信息，不要被贴纸和泡泡完全盖住。

如果用户给了参考照片，尽量保持原照片透视和现场感，只加鼠鼠、泡泡、少量文字和道具。

### 4. 生成或编辑

当用户明确要求“生成 / 做图 / 改图 / 出几张”时，直接使用图像生成或编辑工具，每张图单独生成，不要把多张拼在一张里。

生成前读取 `references/reference-lock.md` 和 `references/prompt-template.md`，并把提示词具体化到当前场景。不要只用抽象风格词；必须明确“照片头 + 简笔白身体 + 粗黑描边 + 手绘云泡泡 + 低清 P 图感”。改图时优先保留原照片、原鼠鼠位置和原构图，只替换用户指定内容。

### 5. 检查与迭代

生成后按 `references/qa-checklist.md` 检查。以下情况要重生成或改图：

- 鼠鼠太像精致吉祥物，不像粗糙梗图贴纸。
- 没有幻想云，或云泡泡不清楚。
- 画面太商业、太干净、太像儿童绘本。
- 中文太多或像正式说明。
- 真实背景被破坏，现场感丢失。
- 鼠鼠和泡泡遮住关键人物、屏幕或主体。

## 输出口径

策略输出要短，优先给 shot list：

- 图名
- 场景
- 鼠鼠位置
- 幻想云内容
- 建议文字
- 生成提示词要点

生成后的交付说明只写生成数量、每张用途和保存路径。不要长篇解释风格理论。
